Pourquoi commit le film avant les acteurs ?
Parce que les Actors dépendent de l'id du Movie (clé étrangère movie_id). Si on ne commit pas d’abord le film, son ID n’est pas encore généré (car auto-incrémenté par la base), donc on ne peut pas créer les acteurs qui y font référence.

Différence entre lazy loading et eager loading (joinedload) ?
Lazy loading : Les données liées (e.g., acteurs) sont chargées seulement quand on y accède. Risque de multiples requêtes SQL (N+1 queries).

Eager loading (joinedload) : Les données liées sont chargées immédiatement avec la requête principale via une jointure SQL. Meilleur pour performance si on sait qu’on va en avoir besoin.

 Comment formater une liste d'acteurs pour une requête LLM ?
